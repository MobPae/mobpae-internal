# Database Migrations

**Tool:** Prisma Migrate  
**Location:** `mobpae-backend/prisma/migrations/`

---

## Workflow

Migrations are generated by Prisma whenever the schema changes. Each migration is a timestamped folder containing a `migration.sql` file.

```bash
# Create a new migration from schema changes
npx prisma migrate dev --name <description>

# Apply migrations in production
npx prisma migrate deploy

# Reset database and reseed (dev only — destructive)
npx prisma migrate reset

# Seed after migrate
npx ts-node prisma/seed.ts
```

---

## Migration Naming Convention

Migrations follow the pattern: `YYYYMMDDHHMMSS_<description>`

Examples:
- `20240101000000_initial_schema`
- `20240115120000_add_employer_risk_status`
- `20240201090000_add_loan_product_config`

## Key Schema Evolution Points

### Foundation
Initial schema created all core models: `User`, `Employer`, `Employee`, `LoanApplication`, `Repayment`, `Disbursal`, `KycDocument`, `EmployeeBankAccount`.

### Product Config System
`LoanProduct`, `LoanProductConfig`, `EmployerProductConfig`, and `LoanLimit` were added to replace hardcoded eligibility/pricing logic. This enabled the versioned config approach where rates can change without code deploys.

### Settlement System
`EmployerSettlement`, `SettlementLineItem`, `SettlementPayment` added as a unit. Settlement is separate from repayment — repayment tracks what employee owes, settlement tracks what employer owes MobPae.

### Platform Fee
`LoanApplicationFee` added to replace the old membership fee model. The `feeType` column is `PLATFORM_FEE`; the membership concept was removed entirely.

### Funding Partner
`FundingPartner` model added as a placeholder for future NBFC integration. All existing applications have `fundingPartnerId = NULL` indicating self-funding.

### Snapshot Fields
`LoanApplication` gained all `snapshot*` fields in a batch migration. Previously rates were read from config at read-time; snapshots freeze them permanently at submission time.

### Selfie Verification Removal
Selfie identity verification (`Employee.selfieUrl`/`selfieStatus`/`selfieVerifiedAt`/`selfieVerifiedBy`, `SelfieStatus` enum, `LoanProductConfig.eligibilityRules.requiresActiveSelfie`) was removed from the product — it was never surfaced in the admin panel and never implemented in the employee app. Migration `20260717120000_remove_selfie_verification` drops the columns, index, and enum. **Not applied automatically** — written for manual review before running `prisma migrate deploy`.

---

## Production Notes

- **Never edit migration files after they've been applied.** If you need to fix a mistake, create a new migration.
- **Never use `prisma migrate reset` in production.** It drops and recreates the database.
- **`prisma migrate deploy`** is the only command safe for production — it applies pending migrations in order without resetting.
- **Seed data** (`prisma/seed.ts`) is for development only. Do not run the seed in production.
